# Excel Brain (eb)

에이전트에게 **무한·구조화된 기억**을 주는 타입 지식 그래프. 지식은 마크다운이 아니라 **표(스프레드시트) 묶음**에 살고, 그래프 탐색은 결정적(deterministic) Python 엔진이 보증한다. [이 영상](https://www.youtube.com/watch?v=z02Y-1OvWSM)에서 영감을 받았다.

## Language

**노드 (Node)**:
하나의 원자적 아이디어. 타입(pillar/decision/concept/fact 등)을 가진다.
_Avoid_: 노트, 문서, 카드

**엣지 (Edge)**:
두 노드 사이의 **방향 있는·타입 있는·이유 있는** 관계.
_Avoid_: 링크, 연결, 참조

**타입 그래프 (Typed graph)**:
모든 노드와 엣지가 각자 타입을 갖는 지식 그래프. eb가 다루는 지식의 형태.
_Avoid_: 노트 더미, 위키, 트리

**지식의 원천 (Source of truth)**:
모든 지식이 사는 단일하고 권위 있는 스프레드시트 묶음. 다른 모든 표현(시트, 캐시 DB)은 여기서 파생된 **뷰**일 뿐이다.
_Avoid_: DB, 백업, 저장소

**증류 (Distill)**:
원자료(텍스트·대화)를 노드와 엣지로 쪼개어 그래프에 편입시키는 행위. `eb-capture`가 하는 일.
_Avoid_: 임포트, 붙여넣기, 저장

**조회 (Recall)**:
그래프를 탐색해 답을 얻는 행위(이웃·경로·요소). `eb-recall`이 하는 일.
_Avoid_: 검색, 쿼리, 룩업

**정제 (Curate)**:
그래프의 건강을 회복시키는 행위 — 중복 병합, 고아 연결, 드리프트 교정. `eb-curate`가 하는 일.
_Avoid_: 리팩터, 청소, 정리

**신뢰도 (Confidence)**:
한 노드의 내용을 얼마나 믿을 수 있는가(0.0~1.0). 시간이 지나며 갱신될 수 있는 메타데이터.
_Avoid_: 점수, 우선순위, 가중치

**연결 제안 (Suggest)**:
기존 그래프 구조(공통 이웃·태그 겹침·2홉)로부터 "이 노드는 저 노드에 붙어야 한다"를 결정적으로 계산해 내놓는 후보. 캡처와 정제가 모두 사용한다.
_Avoid_: 추천, 힌트, 자동완성

**그래프-인지 캡처 (Graph-aware capture)**:
지식을 추가하기 *전에* 그래프를 먼저 조회해 중복을 피하고 연결처를 찾는 캡처 방식. eb의 핵심 차별점.
_Avoid_: 자동 추가, 임포트

**고아 노드 (Orphan)**:
어떤 엣지와도 연결되지 않아 그래프에서 도달 불가능한 노드. 정제의 주요 대상.
_Avoid_: 미아, 데드노드

**드리프트 (Drift)**:
원천(CSV)과 파생 뷰(시트)가 어긋난 상태. 뷰는 원천으로 되돌려야 한다.
_Avoid_: 불일치, 충돌, 디싱크
