# 그래프-인지 캡처와 엔진 보강(search/merge/suggest)

`eb-capture`는 노드를 곧장 만들지 않고 추가 *전에* `eb.py`로 기존 그래프를 탐색한다 — ① 중복 후보 탐지 ② 새 지식이 붙을 기존 노드(엣지 타깃) 제안 → 사용자 승인 → `add-node`/`add-edge` → `validate`. 이를 위해 `eb.py`에 세 명령을 추가한다: `search`(유사 노드 검색), `merge`(중복 노드 엣지 재배선 후 삭제), `suggest`(공통 이웃·태그 자카드·2홉 등 그래프 구조로 연결 후보 제안). 모두 stdlib only를 유지한다.

## 왜

eb의 차별점은 ib와 달리 그래프 연산을 프롬프트가 아니라 **결정적 Python 엔진**이 보증한다는 것이다. 이 원칙을 *읽기*뿐 아니라 *쓰기*에도 적용해, 캡처가 고아·중복을 만들지 않고 그래프를 촘촘하게 키우도록 한다(`suggest`가 그 정수).

## Consequences

- `suggest`는 `eb-capture`(붙일 곳 찾기)와 `eb-curate`(그래프 촘촘화) 양쪽에서 재사용된다.
- 자동 커밋이 아닌 **승인 게이트**가 기본이라, 저마찰보다 정확성을 우선한다.
