# 이비서 설정 매뉴얼 — 컴퓨터 일꾼(에이전트) 켜기

> 이비서의 **일꾼(워처)** 은 마법이 아니라 **내 컴퓨터에서 켜둔 AI 에이전트**(Claude Code · Codex · Antigravity 중 하나)다.
> 폰 제미나이가 "작업 메모판(`<이름>-queue` 캘린더)"에 적어둔 일을, 이 컴퓨터 일꾼이 **몇 분마다 들여다보며 대신 처리**한다.
> 처음 한 번만 이 일꾼을 깔고 켜면 된다. 비개발자도 된다. 막히는 줄은 복사해서 그 에이전트에게 "이거 해줘"라고 물어도 된다.

---

## 한눈에

```
[나] --말로--> [폰 제미나이] --못 하는 일은 메모--> [<이름>-queue 캘린더] <--몇 분마다-- [내 컴퓨터의 에이전트 = 일꾼]
                                                                                          └ 안전하게 처리 → 메모에 완료 도장
```

- **이름**: 비서 하나당 base 이름 `<이름>` 을 정한다(기본 `eb1`). 지식 시트 = `<이름>`(eb1), 작업큐 캘린더 = `<이름>-queue`(eb1-queue). 한 사람이 **여러 개**(eb1, eb2, 팀용 `team1` …) 운영 가능 — 이미 있으면 셋업이 다음 번호를 제안한다.
- **일정**(생성·조회·수정·삭제)은 제미나이가 **직접** 한다. 큐로 가는 건 메일 발송·시트/문서 변경·지식 추가 같은 **되돌리기 어려운 일**뿐.

---

## 준비물

컴퓨터 1대(Windows/Mac) · 본인 **개인 Google 계정** · 폰(Android/iPhone) · 인터넷.

---

## 0단계 — 어떤 에이전트를 쓸지 고르기

일꾼은 셋 중 하나로 돌린다. **처음이라면 Claude Code 를 추천**(가장 검증됨 + 브라우저로 셋업 도와줌).

| 에이전트 | 설치 | 4분 워처 | 24/7 | 브라우저 셋업 |
|---|---|---|---|---|
| **Claude Code** (추천) | docs.anthropic.com/en/docs/claude-code | 세션 cron 또는 **클라우드 루틴(`/schedule`)** | 클라우드 루틴이면 PC 꺼도 OK | 가능(로그인된 크롬 전제, 반자동) |
| **Codex** (OpenAI) | `npm i -g @openai/codex` 등 | **OS cron/작업 스케줄러** → `codex exec` | 로컬뿐(PC 켜둬야) | 무인 부적합 → 폼은 사람이 |
| **Antigravity** (Google) | Antigravity 설치 | `/schedule` 또는 OS cron → `agy` 헤드리스 | 로컬뿐 | **풀자동 강점**(내장 브라우저) |

> Codex/Antigravity는 공개 프리뷰라 명령·플래그가 바뀔 수 있다. 켜기 전 `codex --help` / `agy --help` 로 현행을 확인.

아래는 **Claude Code 기준**으로 쓰되, 다른 에이전트는 4단계 표의 해당 줄로 바꾸면 된다.

---

## 1단계 — 에이전트 설치 + 폴더 열기

1. 위 표의 안내대로 에이전트를 설치하고 로그인한다.
2. 이비서가 살 **빈 폴더**를 하나 만든다(예: 바탕화면 `eb1`). 그 폴더를 에이전트로 연다.
   - 입력 위치 구분: **셸/터미널** 에는 `npx …` 같은 명령, **에이전트 채팅창** 에는 `/eb-…` 명령.

---

## 2단계 — 이비서 깔기 (스킬 + 기본 셋업)

**터미널**:
```bash
npx skills@latest add jhs512/eb --all
```
**에이전트 채팅창** (각 단계는 에이전트가 손잡아 안내):
```
/eb-setup     # 기억 보드 만들기 — 맨 처음 비서 이름을 물어본다(기본 eb1)
/eb-gcp       # (Google 계정당 1회) 시트에 쓸 열쇠
/eb-sheets    # 내 기억을 '<이름>' 구글 시트로 비추기 (폰 지식조회의 핵심)
/eb-github    # 개인(비공개) 백업 + 자동화
/eb-pages     # (선택) 웹페이지로 보기 (Cloudflare, 무료) — README 아기단계 참고
```
> 이름 기본값: 시트 `eb1`, 큐 캘린더 `eb1-queue`. 다른 이름(예 `mye`)도 된다.

---

## 3단계 — 작업큐 캘린더 만들기

작업 메모판인 **보조 캘린더 `<이름>-queue`**(예 `eb1-queue`)가 필요하다.
- ⚠️ 보조 캘린더 **생성은 PC 구글 캘린더 웹에서만** 된다(폰 앱 불가). [Google Calendar](https://calendar.google.com) → 좌측 **"다른 캘린더 +"** → **새 캘린더** → 이름 `eb1-queue`.
- 평소엔 사이드바 체크를 꺼 안 보이게, **아무에게도 공유 안 함**(이 비공유가 신뢰 경계).
- **브라우저 자동화가 되는 에이전트**(Antigravity, 또는 로그인된 크롬의 Claude Code)면 "캘린더 만들어줘"로 **대신 띄워 만들 수** 있다. 안 되면 위처럼 직접.

---

## 4단계 — 일꾼(워처) 켜기 ★ 핵심

에이전트 채팅창에 `/eb-queue`. 이게 큐 캘린더를 확인·등록하고 **워처**를 켠다. 등록 방법은 에이전트별로:

| 에이전트 | 워처 켜는 법 |
|---|---|
| **Claude Code** | 세션 cron(이 창이 떠 있는 동안) **또는** `/schedule` 클라우드 루틴(PC 꺼도 돎, **권장**) |
| **Codex** | OS cron `*/4 * * * *` 또는 Windows 작업 스케줄러로 `codex exec "<틱 프롬프트>"` 를 4분마다 |
| **Antigravity** | `/schedule` 에 cron `*/4 * * * *`, 또는 OS cron → `agy` 헤드리스 |

켜지면 화면에 몇 분마다 상태 한 줄: `대기 0 · 실행중 0 · 완료 0 · 실패 0 · 확인필요 0 | 오늘 처리 0`
(**확인필요** = 위험해서 당신 승인을 기다리는 일, 예: 메일 발송.)

> **단일 일꾼만** 켠다. 클라우드 루틴과 로컬 일꾼을 **둘 다** 켜면 충돌한다 — 하나만.

---

## 4.5단계 — 컴퓨터 24시간 안 꺼지게 (로컬 일꾼이면 필수)

- **클라우드 루틴으로 켰으면 이 단계는 건너뛴다**(클라우드가 호스팅).
- **로컬 일꾼**(세션 cron · Codex · Antigravity)이면 PC가 24시간 깨어 있어야 한다.

**Windows 11** (관리자 권한, Git Bash면 앞에 `MSYS_NO_PATHCONV=1`):
```bash
powercfg /change standby-timeout-ac 0      # 대기 안 함
powercfg /change hibernate-timeout-ac 0    # 최대절전 안 함
powercfg /change disk-timeout-ac 0
powercfg /setacvalueindex SCHEME_CURRENT SUB_BUTTONS LIDACTION 0 && powercfg /setactive SCHEME_CURRENT   # 노트북 덮개 닫아도 안 잠
```
**macOS**: `caffeinate -dimsu` (또는 `sudo pmset disablesleep 1`). **Linux**: `systemd-inhibit`.

추가로:
- **Windows Update 자동 재부팅** 막기 → 설정 > Windows Update > 고급 옵션 > **활성 시간** 지정.
- **정전 대비** → UPS 또는 정전 후 자동 켜짐.
- **부팅 시 자동 재시작** → 재부팅하면 로컬 일꾼이 죽으므로, 컴퓨터가 켜질 때 에이전트·워처가 자동으로 다시 뜨게 해둔다.
- ⚠️ **세션 cron은 에이전트 창이 떠 있어야 돈다** — PC가 안 꺼져도 창을 닫으면 멈춘다. 진짜 무인 24/7은 결국 **클라우드 루틴**이 답.

## 5단계 — 폰에서 말로 부리기

1. **앱** — Play 스토어/App Store에서 `Google Gemini` 설치, **같은 개인 Google 계정** 로그인.
2. **연결 앱** — 제미나이 설정 → **Google Calendar(필수)·Drive(필수, 지식조회)·Gmail(선택, 초안)** 켜기.
3. **비서 지침(Saved info)** — 제미나이 설정 → Personal Intelligence → **Instructions for Gemini** → **Add +** → 지침 붙여넣기 → **Submit**.
   - 붙여넣을 지침은 `/eb-queue` 가 당신 인스턴스 값(`<이름>`/`<이름>-queue`)으로 채워 준다. 비공개 저장소면 `gemini-saved-info.md` 에 정본이 보관된다.
   - ⚠️ 음성(Gemini Live)엔 이 지침이 적용 안 될 수 있고, 음성에선 시트 조회·Gmail·일정 삭제가 막혀 있을 수 있다 → 안 되면 타이핑으로.

## 이제 폰에 말하세요

- "다음 주 월요일 회의 화요일로 옮겨줘" → 일정이라 제미나이가 **바로**.
- "지난 회의 내용 이비서에 기억해둬" → 큐로 → 일꾼이 처리.
- "그 메일 답장 보내줘" → 큐 → **확인필요**(당신이 직접 승인·발송).
- "작업큐 지금 어때?" → 대기·완료·확인필요 요약.

## 자주 막히는 곳

- **상태 줄이 안 찍혀요/처리가 안 돼요** → 일꾼(에이전트) 창이 켜져 있는지. 닫혔으면 폴더에서 다시 실행. 오래 자리 비우면 클라우드 루틴으로.
- **폰 "내 기억" 조회가 비어요** → 제미나이 **Drive 연결** + 시트 이름이 **`<이름>`(eb1)** 인지.
- **메모판이 안 보여요** → 보조 캘린더라 숨겨둠. 구글 캘린더 좌측에서 `<이름>-queue` 체크를 켜면 보임. 없으면 PC 웹에서 생성(3단계).
- **위험한 걸 시켰는데 안 해요** → 정상. 메일 발송·(비캘린더)삭제 같은 비가역은 자동으로 안 하고 `확인필요`로 당신 승인을 기다린다.
